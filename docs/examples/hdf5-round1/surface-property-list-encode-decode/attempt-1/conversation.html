h5_plist_encode_decode_fuzzer targets the property-list binary parser directly:

  1. H5Pdecode(data) — feeds raw fuzzer bytes as the encoded blob. This is the primary attack surface: any length, any content is a valid test case. Failure is handled gracefully.
  2. H5Pencode2(plist_id, NULL, &size, H5P_DEFAULT) — size query, then fill — exercises the encoder on whatever the decoder produced, covering round-trip consistency bugs.

  4. H5Pclose — always called on success to avoid leaks.

API contracts honored: cd_nelmts initialized before each H5Pget_filter2, name buffer sized to namelen, filter loop guarded on nfilters > 0, fapl_id = H5P_DEFAULT (documented valid).
