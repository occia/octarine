Now I have enough information to write the harness. Let me understand the target API:

  1. H5Sdecode(const void *buf) - takes a buffer and reconstructs a dataspace. This is what we're primarily fuzzing.
  2. H5Sencode2(hid_t obj_id, void *buf, size_t *nalloc, hid_t fapl) - encodes a dataspace to a buffer.
  3. H5Sget_select_type(hid_t spaceid) - gets the selection type (ALL, NONE, POINT, HYPERSLAB)
  4. H5Sis_simple(hid_t space_id) - checks if dataspace is simple
  5. H5Sget_simple_extent_dims(hid_t space_id, hsize_t dims[], hsize_t maxdims[]) - gets dimensions
  6. H5Sclose(hid_t space_id) - closes a dataspace

My approach will be to feed the fuzz input to H5Sdecode, then exercise the resulting dataspace with query functions to verify it's valid, and finally perform a round-trip test by encoding and decoding it again.

The harness needs to initialize HDF5 with H5open() first, suppress error output to keep things clean, then decode the fuzz bytes directly since they represent an encoded dataspace blob. This exercises the binary parser in the decode path.
