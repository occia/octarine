Now I have a good understanding of the API. Let me also check if there's a pattern for the H5Literate function to iterate over datasets in an HDF5 file, since the fuzzer will receive arbitrary HDF5 files.

Let me now write the harness. The strategy:

  1. Write fuzz input to a temp file
  2. Open with H5Fopen (RDONLY)
  3. Iterate over objects in the file using H5Literate2 or H5Ovisit3
  4. For each dataset found:
    • Open with H5Dopen2
    • Get create property list with H5Dget_create_plist
    • Check filters with H5Pget_nfilters and H5Pget_filter2
